\begin{abstract}
Transactional Stream Processing Engines (TSPEs) form the backbone of modern stream applications handling shared mutable states. 
Yet, the full potential of these systems, specifically in exploiting parallelism and implementing dynamic scheduling strategies, is largely unexplored. 
\myc{We present \system, a TSPE designed to optimize parallelism and performance for transactional stream processing on multicores. }
Through a unique three-stage execution paradigm (i.e., \textit{planning}, \textit{scheduling}, and \textit{execution}), \system enables dynamic scheduling and parallel processing in TSPEs. 
\begin{comment}
During Planning, it tackles dependencies using a two-phase \tpg construction process, combined with the notion of virtual operations, thus efficiently handling scenarios of out-of-order transactions and non-deterministic state access. 
During Scheduling, \system dynamically modulates exploration strategy, granularity, and abort handling to adapt to varying input streams and system states. 
During Execution, 
\myc{
\system ensures correct transaction handling through a finite state machine augmented Stateful Task Precedence Graph (\stpg).
}
\end{comment}
