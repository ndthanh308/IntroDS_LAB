

\section{Results}
In this section, we present the performance results of SD algorithms in 11 data sets, including our LC dataset.

\subsection{Subgroup Discovery algorithms}
\label{sec:Results1}
Initially, performance comparison of FSSD, SSD++, and IGSD algorithms has been made based on descriptive and predictive measures. FSSD and SSD++ were used with default parameters, and the two versions of IGSD (i.e. IGSD-M and IGSD-T) were tested according to the two possible values of algorithm argument $t_{mode}$. Also, for the P4Lucat dataset, results for both versions of IGSD were obtained by including cancer stage and first treatment variables in the $Cond_{list}$ argument. In this way, all discovered patterns will contain these variables from the dataset, since clinicians considered them essential variables.

The summary of results in terms of descriptive and predictive measures is as follows, considering that SSD++ algorithm was not able to discover patterns in the ECHO dataset as well as FSSD algorithm regarding the MAGIC dataset:
\begin{itemize}
\item For IRIS dataset, IGSD-M and IGSD-T returns a little more number of discovered patterns (9, 7) with respect to FSSD and SSD++ (5, 5). Otherwise, for ECHO and HEART datasets, IGSD is discovering slightly more patterns (9, 12 and 7, 22 respectively) with respect to FSSD (4, 2) and SSD++ (-, 6). On the other hand, SSD++ discovered much more patterns (92) than IGSD-M and IGSD-T (9, 51).
\item In terms of the complexity of the rule set (length), FSSD produces the set of patterns with the largest number of variables. However, considering that FSSD did not discover patterns for the MAGIC dataset, SSD produces a set of patterns with slightly more number of variables (3.97) than both IGSD versions (3.56, 3.59).
\item Regarding the average coverage per rule, considering the IRIS and HEART dataset, patterns produced by SSD++ have higher values (0.48, 0.29) than the patterns produced by both IGSD versions and SDD++ algorithms with values similar to 0.2. Moreover, in the ECHO dataset, FSSD produced patterns with higher values (0.247) with respect to IGSD-M and IGSD-T (0.1, 0.083). On the other hand, concerning MAGIC dataset, IGSD-M and IGSD-T reports higher values (0.042, 0.039) respecting SDD++ (0.032).
\item Assessing the unusualness of the patterns, considering the IRIS dataset, all the algorithms produce sets of patterns with a value similar to 0.12. Besides, in the ECHO dataset, FSSD produces a set of patterns with a higher value (0.247) than both IGSD versions (0.1, 0.083). On the other hand, concerning HEART and MAGIC datasets, IGSD-M produces slightly higher values (0.089, 0.02) than IGSD-T (0.075, 0.019), and much higher than SDD++ (0.027, 0.0048).
\item When evaluating the rule confidence, regarding ECHO and MAGIC datasets, the sets of patterns produced by IGSD-M (0.91, 0.96) and IGSD-T (0.9, 0.95), have more confidence values than FSSD (0.83, -) and SSD++(-, 0.74). Besides, regarding IRIS dataset, IGSD-T produces a set of patterns with slightly more confidence value (0.98) than IGSD-M and FSSD (0.94, 0.96, respectively) and much higher than SSD++ (0.67). Moreover, concerning HEART dataset, IGSD-M and IGSD-T set of patterns produces higher values (0.87, 0.86) than FSSD (0.82) and SDD++ (0.66).
\item In terms of accuracy prediction, concerning the IRIS dataset, both IGSD versions and FSSD algorithms might be considered reliable models due to the accuracy values reported, 0.84, 0.84 and 0.85 respectively, meanwhile SSD++ accuracy decreases to a value of 0.73.
Regarding the ECHO dataset, FSSD reports an accuracy value of 0.77, meanwhile, IGSD algorithm reports a lower accuracy value (0.61, 0.65 respectively). On the other hand, respecting HEART and MAGIC datasets, IGSD-M and IGSD-T report a slightly higher accuracy value (0.67, 0.59 and 0.64, 0.59, respectively) than FSSD (0.63, -) and SSD++ (0.55, 0.52) algorithms. 
\item Considering the average IG per rule, concerning the IRIS and ECHO datasets, IGSD-M and IGSD-T report patterns with less information gained (0.327, 0.05) and (0.327, 0.049) respectively than FSSD algorithm, with values of 0.47 and 0.117 respectively. Regarding the HEART dataset, SSD++ reports the highest information gained value (0.174), meanwhile, IGSD-M, IGSD-T and FSSD report lower information gained values (0.137, 0.11 and 0.1 respectively). Furthermore, respecting the MAGIC dataset, IGSD-M and IGSD-T produce sets of patterns with information gained values of 0.036 and 0.034 respectively, meanwhile, SSD++ produces a set of patterns with a lower information gained value of 0.022.
\item Taking into consideration the ORR, concerning the IRIS dataset, patterns produced by both IGSD versions and FSSD report an ORR of 4, meanwhile SSD++ patterns produce a lower ORR of 3.4. Regarding the ECHO dataset, FSSD patterns report an ORR value of 4, meanwhile, both IGSD versions report lower ORR values (3.89, 3.92 respectively). On the other hand, in regard to HEART and MAGIC datasets, IGSD-M and IGSD-T set of patterns report the highest ORR values (4, 4) and (4, 3.98) respectively, meanwhile, FSSD and SDD++ patterns report lower ORR values (3.5, -) and (3, 3.16) respectively.
\item In terms of p-value, regarding the IRIS dataset, both IGSD versions and SDD++ algorithms produce statistically significant patterns due to reporting a p-value below 0.05. Nevertheless, the IGSD algorithm reports a value below 0.001, which indicates that patterns might be considered more statistically significant. On the other hand, the FSSD algorithm reports patterns with a p-value above 0.05. In regards to the ECHO dataset, both IGSD versions and FSSD algorithms produce statistically significant patterns due to reporting a p-value below 0.05, nonetheless, FSSD reports a p-value below 0.01, being able to consider the patterns more statistically significant. Furthermore, concerning the HEART and MAGIC datasets, all the algorithms report a p-value below 0.001, which indicates a strong statistical significance.
\end{itemize}

% Analyzing all metric results enumerated above, it can be seen that FSSD and SSD++ algorithms contain patterns with less complexity, hence, less information, than both IGSD algorithms. Patterns from SSD++ algorithm especially contain very little complexity and amount of information. Thus, the low complexity results in having more representation in the dataset, and having a higher coverage. However, SSD++ patterns are less reliable due to the fewer confidence values reported. Also, FSSD and SSD++ confidence values are 49\% and 31\%, respectively. So, they are significantly lower than the confidence obtained by both versions of IGSD algorithm with a value above 80\%. This low reliability is also reflected in ORR values, with FSSD having 2.84, which represents low dependence between the patterns and the targets, and SSD++, 1.69, which represents even a lower dependence between the patterns and the targets. However, the ORR reported by both versions of IGSD (3,84-3,88) represents better reliability of the patterns with a medium-high dependence between patterns and targets. \\

The summary of results in terms of descriptive and predictive measures is as follows:
\begin{itemize}
\item Regarding the TIC-TAC-TOE dataset, SSD++ returns more number of discovered patterns (17) with respect to FSSD (10) and both IGSD versions (4). On the other hand, concerning the VOTE and P4Lucat dataset, IGSD-T was able to discover a higher amount of patterns (21, 52) in contradistinction to IGSD-M (13, 19) and FSSD (10, 20) and SSD++ (4, 16). Moreover, considering the GENBASE dataset, both IGSD versions produce a huge amount of patterns (37041, 45625), as opposed to FSSD and SSD++ algorithms (32, 33).

\item In terms of the complexity of the rule set (length), regarding the TIC-TAC-TOE and P4Lucat, both IGSD versions produce sets of patterns with the largest number of variables (3, 3.1) and (3, 3.9). On the contrary, SSD++ sets of patterns have the least complexity (2.29, 1.11), thus containing much less information.
Furthermore, respecting VOTE and GENBASE datasets, FSSD sets of patterns have a much more number of variables (7.7, 1151) than IGSD-M (1.46, 2), IGSD-T (2.14, 2) and SSD++ (2.5, 1) respectively.

\item Regarding the average coverage per rule, considering the TIC-TAC-TOE dataset, patterns produced by FSSD have the highest coverage value (0.13), while both IGSD versions sets of patterns have less coverage value (0.073). Furthermore, considering the VOTE dataset, IGSD-M reports a slightly higher value (0.41) than IGSD-T (0.37), meanwhile, FSSD returns a set of patterns with much less coverage (0.12). Finally, with respect to P4Lucat and GENBASE datasets, SSD++ produces a set of patterns with a high coverage value (0.35, 0.128), meanwhile, FSSD and both IGSD versions report a much lower coverage value, (0.094, 0.032) and (0.024, 0.0086 and 0.016, 0.027) respectively.

\item Assessing the unusualness of the patterns, regarding the TIC-TAC-TOE dataset, FSSD and both IGSD versions produce sets of patterns with high unusualness value similar to 0.035, however, SSD++ set of patterns has lower unusualness value (0.02). Moreover, concerning the VOTE dataset, IGSD-M and IGSD-T report higher values (0.173, 0.15) than FSSD and SSD++ (0.05, 0.085 respectively). Finally, respecting P4Lucat and GENBASE datasets, FFSD produces patterns with higher unusualness value (0.017, 0.027) than IGSD-M (0.0095, 0.008), IGSD-T (0.0068, 0.02) and SSD++ (0.011, 0.02).

\item When evaluating the rule confidence, concerning the TIC-TAC-TOE and P4Lucat datasets, IGSD-M and IGSD-T report higher confidence values (1, 0.83) and (1, 0.81) respectively, however, in P4Lucat dataset, FSSD and SSD++ report much lower confidence values (0.51, 0.31). Furthermore, regarding the VOTE dataset, the set of patterns discovered by FSSD reports a confidence value of 0.98, meanwhile, IGSD-M and IGSD-T confidence values are slightly lower (0.91, 0.9) and SSD++ confidence value are much lower (0.75). Finally, with respect to GENBASE dataset, both IGSD versions and FSSD algorithms report high confidence values similar to 0.98 in contrast with the low confidence value reported by SSD++ (0.348).

\item In terms of accuracy prediction, regarding the TIC-TAC-TOE dataset, all algorithms report an accuracy value similar to 0.57. However, with respect to VOTE and P4Lucat datasets, both IGSD versions and FSSD report a high accuracy value, being above 0.8 in VOTE dataset for IGSD algorithm and above 0.7 in P4Lucat for the mentioned algorithms, thus might be considered reliable models. Finally, considering the GENBASE dataset, all the algorithms can be considered very reliable models due to reported accuracy values above 0.9.

\item Considering the average IG per rule, with respect to TIC-TAC-TOE and VOTE datasets, IGSD-M and IGSD-T report high IG values (0.072, 0.45) and (0.072, 0.33) respectively and SSD++ reports a high IG value considering the VOTE dataset (0.41). Furthermore, regarding the P4Lucat dataset, SSD++ was able to discover patterns with an IG value of 0.034, meanwhile, both IGSD versions and FSSD report slightly lower values (0.016, 0.011, 0.019). Finally, considering the GENBASE dataset, FSSD, SSD++ and IGSD-T produce sets of patterns with similar IG value (0.1, 0.15, 0.1), meanwhile IGSD-M set of patterns has a lower IG value of 0.048.

\item Taking into consideration the ORR, patterns produced by IGSD-M and IGSD-T report the highest ORR value for all the datasets. On the other hand, FSSD reports an ORR value slightly lower in TIC-TAC-TOE dataset, and a significantly lower ORR value regarding P4Lucat.
In addition, although SSD++ reports the same ORR value as the rest of the algorithms considering the GENBASE dataset, regarding the other datasets SSD++ reports a considerably lower ORR value.

\item In terms of p-value, regarding TIC-TAC-TOE and GENBASE  datasets, both IGSD versions and FSSD algorithms produce highly statistically significant patterns due to reporting a p-value below 0.001, meanwhile SSD++ reports a p-value above 0.05 in TIC-TAC-TOE dataset, thus the set patterns could be considered statistically irrelevant, meanwhile reports a p-value below 0.05 in GENBASE dataset. Furthermore, concerning the VOTE dataset, both IGSD versions and SDD++ algorithms produce highly statistically significant patterns due to reporting a p-value below 0.001, meanwhile, FSSD reports a p-value above 0.05, thus the set patterns could be considered statistically irrelevant. Finally, with respect to P4Lucat, both IGSD versions report a p-value below 0.01, meanwhile, FSSD reports a p-value below 0.05 and SDD++ reports a p-value above 0.05.
\end{itemize}

The summary of results in terms of descriptive and predictive measures is as follows:
\begin{itemize}
\item Regarding the BREAST-CANCER dataset, FSSD returns more number of discovered patterns (10) with respect to both IGSD versions and SSD++ algorithms, which return a similar amount of patterns (4, 5, 3 respectively). Furthermore, concerning the Nursery dataset, SSD++ was able to discover a higher amount of patterns (97) in contradistinction to IGSD versions (13, 20) and FSSD (13).
Finally, with respect to the HEART dataset, IGSD-T discover more patterns (22) than IGSD-M and FSSD and SSD++ algorithms (7, 2, 6 respectively.)
\item In terms of the complexity of the rule set (length), FSSD produces sets of patterns with the largest number of variables for the 3 datasets. However, it can be noticed that regarding the HEART dataset, FSSD set of patterns has a much more number of variables (13) than both IGSD versions (1.86, 2.14) and SSD++ (1.83).
\item Regarding the average coverage per rule, considering the BREAST-CANCER and Nursery datasets, patterns produced by FSSD have slightly higher values (0.11, 0.1 respectively) than IGSD-M (0.072, 0.063), IGSD-T (0.072, 0.08) and SSD++ (0.1, 0.06). On the contrary, in the HEART dataset, SSD++ reports the highest coverage value (0.29), followed by both IGSD versions (0.257, 0.22) and FSSD (0.2).
\item Assessing the unusualness of the patterns, IGSD-M produces sets of patterns with the highest unusualness values (0.03, 0.039, 0.089). Moreover, IGSD-T and FSSD report slightly lower values (0.03, 0.033, 0.075) and (0.02, 0.03, 0.07) respectively, meanwhile SSD++ reports much lower values compared to the others (0.0085, 0.01, 0.027).
\item When evaluating the rule confidence, IGSD-M produces sets of patterns with the highest confidence values (0.91, 0.87, 0.87). Furthermore, IGSD-T and FSSD report slightly lower values (0.82, 0.72, 0.86) and (0.82, 0.64, 0.82) respectively, meanwhile SSD++ reports much lower values compared to the others (0.65, 0.63, 0.66).
\item In terms of accuracy prediction, regarding the NURSERY dataset, all the algorithms might be considered reliable models due to the reported accuracy values (0.78, 0.78 and 0.74, and 0.73 respectively). Respecting the BREAST-CANCER and HEART datasets, the accuracy values decrease to values similar to 0.6, although SSD++ reports low accuracy values (0.46, 0.55 respectively). 
\item Considering the average IG per rule, SSD++ was able to discover patterns in BREAST-CANCER and HEART datasets with the highest IG value (0.065, 0.174), meanwhile IGSD-M and IGSD-T discover patterns with slightly less IG values (0.53, 0.53) and (0.137, 0.11) respectively. On the contrary, FSSD reports the lowest IG values, above all in the BREAST-CANCER dataset with an IG value of 0.024. 
\item Taking into consideration the ORR, patterns produced by IGSD-M report the highest ORR value (4 for all the datasets), meanwhile, patterns produced by IGSD-T have ORR values slightly less regarding the Breast-Cancer and NURSERY datasets. On the other hand, FSSD and SDD++ algorithms report lower ORR values regarding all the datasets (3.1, 3.15, 3.5) and (3, 2.7, 3) respectively.  
\item In terms of p-value, regarding the BREAST-CANCER dataset both IGSD versions and SDD++ algorithms produce statistically significant patterns due to reporting a p-value below 0.01. On the other hand, the FSSD algorithm reports patterns with a p-value above 0.05. 
Furthermore, concerning the NURSERY and HEART datasets, all the algorithms report a p-value below 0.001, with the exception of SSD++ algorithm in the NURSERY dataset, reporting a p-value above 0.001; which indicates a strong statistical significance; with the exception of SSD++ algorithm in the NURSERY dataset, which reports a p-value above 0.001.
\end{itemize}

Analyzing all metric results exposed above, it can be seen that patterns generated by IGSD and SSD++ have similar complexity values, in contradiction to FSSD, producing patterns with more complexity and amount of information.
However, considering the nominal TIC-TAC-TOE and P4Lucat datasets, IGSD was able to produce patterns with more complexity than FSSD and SSD++. Besides, the sets returned by IGSD usually contain more patterns than FSSD and SSD++. Thus it can be concluded that IGSD produces patterns with less amount of information but in greater quantity than the other algorithms. Moreover, considering the coverage measure, it exists variability among different dataset types and different datasets, so IGSD can discover patterns with considerable representation in datasets as well as FSSD and SSD++. Furthermore, with respect to coverage and ORR measures, it can be noticed that as a general rule, IGSD patterns are more reliable due to reported values, being the patterns produced by SSD++ the least reliable. In addition, 
the p-value of the patterns produced by IGSD are always below 0.05, being in several datasets above 0.001, thus these patterns can be considered as highly statistically significant. On the other hand, FSSD and SSD++ can not guarantee statistically significant patterns due to the p-values above 0.05 reported in IRIS, TIC-TAC-TOE, VOTE, P4LUCAT, and BREAST-CANCER datasets. Therefore, in summary, it can be concluded that IGSD is able to discover a considerable number of statistically significant patterns with also a high dependence on targets, but offering less amount of information than FSSD.
Otherwise, it is noticeable that considering numeric datasets such as MAGIC and ECHO, FSSD and SSD++, being executed with default values for the corresponding parameters, have found issues to discover patterns in the search space, meanwhile, IGSD was able to finish and uncover different patterns. This limitation could be due to the way FSSD and SSD++ handle the generation of ranges using numeric data.
Moreover, regarding the GENBASE dataset, IGSD has shown different problems regarding the finish of the exploration phase in a time interval of one hour. This problem can be attached to the number of columns of the dataset, thus, IGSD requires much more time to achieve the discovery of patterns. So, this limitation has resulted in a non-reasonable number of patterns, hindering the posterior patterns analysis.
On the contrary, FSSD and SSD++ were able to complete the exploration phase in the same interval of time, resulting in a decent number of patterns according to the number of targets presented in the dataset.
